## 一、项目目标

打造一个 AI 驱动的 Web 自动化测试平台，核心能力：

1. **🤖 AI 驱动浏览器** — 基于 Playwright + DeepSeek LLM，用自然语言描述测试步骤，AI 自动操控浏览器执行
2. **🔗 禅道深度集成** — 双向同步：从禅道拉取用例 → 自动执行 → 结果/Bug 回写禅道
3. **📝 智能用例管理** — 用例 CRUD、评审、版本管理、自然语言生成可执行步骤
4. **📊 可视化报告** — 通过率统计、失败详情、历史趋势

## 七、关键技术决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 前后端分离 | ✅ | 独立开发、独立部署、技术栈解耦 |
| 开发顺序 | 前端优先 | 先用 Mock 数据完成 UI，后端跟进时无需反复改前端 |
| UI 框架 | Ant Design 5 | 企业级中后台标准，表格/表单/通知组件开箱即用 |
| 状态管理 | Zustand | 比 Redux 简洁，比 Context 性能好 |
| 后端框架 | FastAPI | 异步性能好，自动生成 API 文档，类型提示友好 |
| AI 引擎 | Playwright sync + LLM（自研） | 原选 browser-use，因 Python 3.14 + Windows 下 `asyncio.create_subprocess_exec` 死锁，改为 sync Playwright + LLM 自主决策执行 |
| 浏览器引擎 | Playwright | 比 Selenium 更快，原生支持多浏览器 |
| 禅道集成方式 | REST API + Apache 基本认证 | ZenTao Box 使用 Apache 反向代理，需先过 HTTP 基本认证再调 REST API |
| Celery 运行模式 | `--pool=solo` | 避免 Python 3.14 + Windows 的多进程子进程问题 |
| 前后端联调方案 | Vite 代理 + 响应转换 | 前端 `/api/v1` → Vite 代理 → `localhost:8000`；拦截器统一解包和驼峰转换 |
